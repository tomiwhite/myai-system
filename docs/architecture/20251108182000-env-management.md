## 📁 Hierarchia Plików .env

### 1️⃣ MASTER `.env` (Per User)
```
📍 Lokalizacja: C:\Users\tomiw\.env
🔒 Poziom: GLOBAL
⚡ Priorytet: 1 (najwyższy)
```

**Zawiera**:
- Poufne klucze API (OpenAI, Azure, GitHub)
- Hasła do baz danych
- Tokeny autoryzacyjne
- Subskrypcje

**Przykład**:
```env
# MASTER .env - C:\Users\tomiw\.env
# Poufne dane - NIGDY NIE COMMITOWAĆ

# AI APIs
OPENAI_API_KEY=sk-proj-xxxxxxxxxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
AZURE_OPENAI_KEY=xxxxxxxxxxxx

# Cloud
AZURE_SUBSCRIPTION_ID=xxxxxxxxxxxx
AWS_ACCESS_KEY_ID=xxxxxxxxxxxx
AWS_SECRET_ACCESS_KEY=xxxxxxxxxxxx

# Databases
DATABASE_PASSWORD=xxxxxxxxxxxx
REDIS_PASSWORD=xxxxxxxxxxxx

# GitHub
GITHUB_TOKEN=ghp_xxxxxxxxxxxx
```

**Backup**: `C:\myAI_System\backups\config-backups\.env.master.backup`

---

### 2️⃣ APPLICATION `.env` (Per Project)
```
📍 Lokalizacja: D:\myai\.env
🌐 Poziom: APPLICATION
⚡ Priorytet: 2
```

**Zawiera**:
- Konfiguracja projektu D:\myai
- Ścieżki do shared resources
- Ustawienia logowania
- Porty i endpointy

**Przykład**:
```env
# APPLICATION .env - D:\myai\.env
# Można commitować (bez sekretów)

# Environment
ENVIRONMENT=development
DEBUG=true

# Application

# Paths
SHARED_CONFIG_PATH=D:\myai\shared\config
SHARED_TOOLS_PATH=D:\myai\shared\tools

# Logging
LOG_LEVEL=INFO
LOG_PATH=D:\myai\logs

# Model Settings

DEFAULT_TEMPERATURE=0.7
DEFAULT_MAX_TOKENS=4000

# System Integration
SYSTEM_LAYER_PATH=C:\myAI_System
MONITORING_ENABLED=true

# Placeholders (set in MASTER .env)
OPENAI_API_KEY=<set-in-C:\Users\tomiw\.env>
GITHUB_TOKEN=<set-in-C:\Users\tomiw\.env>
```

**Backup**: `C:\myAI_System\backups\d-drive-configs\.env.application.backup`

### 3️⃣ AGENT `.env.local` (Per Agent)
```
📍 Lokalizacja: D:\myai\agents\{agent-name}\.env.local
🤖 Poziom: AGENT
⚡ Priorytet: 3 (nadpisuje wcześniejsze)
```

**Zawiera**:

**Backup**: Opcjonalne (tylko dla agentów produkcyjnych)

## ⚙️ Jak Agenci Ładują `.env`

### Python Implementation

```python
# agent_config.py
import os
from pathlib import Path
from dotenv import load_dotenv

class ConfigLoader:
    """Ładuje konfigurację z hierarchii .env"""
    
    def __init__(self, agent_name: str = None):
        self.agent_name = agent_name
        self.load_all()
    
    def load_all(self):
        """Ładuje wszystkie .env w kolejności priorytetów"""
        
        # 1. MASTER (poufne klucze)
        master_env = Path(r"C:\Users\tomiw\.env")
        if master_env.exists():
            load_dotenv(master_env, override=False)
            print(f"✅ Loaded MASTER .env: {master_env}")
        else:
            print(f"⚠️  MASTER .env not found: {master_env}")
        
        # 2. APPLICATION (config projektu)
        app_env = Path(r"D:\myai\.env")
        if app_env.exists():
            load_dotenv(app_env, override=True)
            print(f"✅ Loaded APPLICATION .env: {app_env}")
        
        # 3. AGENT (nadpisania lokalne)
        if self.agent_name:
            agent_env = Path(f"D:/myai/agents/{self.agent_name}/.env.local")
            if agent_env.exists():
                load_dotenv(agent_env, override=True)
                print(f"✅ Loaded AGENT .env: {agent_env}")
    
    def validate_required(self, required_vars: list):
        """Sprawdza czy wymagane zmienne istnieją"""
        missing = []
        for var in required_vars:
            if not os.getenv(var):
                missing.append(var)
        
        if missing:
            raise ValueError(f"Missing required env vars: {missing}")
        
        return True

# Użycie w agencie
config = ConfigLoader(agent_name="conversational-agent")
config.validate_required([
    "OPENAI_API_KEY",
    "SHARED_CONFIG_PATH",
    "LOG_LEVEL"
])

# Teraz wszystkie os.getenv() działają
api_key = os.getenv("OPENAI_API_KEY")
log_level = os.getenv("LOG_LEVEL")
```

## 🔐 Zasady Bezpieczeństwa

### MASTER .env
- ✅ Przechowywać **TYLKO** w `C:\Users\tomiw\.env`
- ✅ Dodać do `.gitignore` globalnie
- ✅ Backup z szyfrowaniem co tydzień
- ❌ **NIGDY** nie logować wartości
- ❌ **NIGDY** nie commitować
- ❌ **NIGDY** nie wysyłać przez email/chat

### APPLICATION .env
- ✅ Można commitować do repo
- ✅ Używać placeholderów dla sekretów
- ✅ Dokumentować wymagane zmienne
- ❌ Nie przechowywać prawdziwych kluczy API

### AGENT .env.local
- ✅ Tylko dla testów lokalnych
- ✅ Dodać `*.env.local` do `.gitignore`
- ❌ Usuwać po zakończeniu developmentu

### Skrypt Backupu (PowerShell)

```powershell
# C:\myAI_System\scripts\backup\backup-env-files.ps1

$timestamp = Get-Date -Format "yyyyMMdd-HHmmss"

# Backup MASTER .env
$masterSrc = "C:\Users\$env:USERNAME\.env"
$masterDst = "C:\myAI_System\backups\config-backups\$timestamp-.env.master"
if (Test-Path $masterSrc) {
    Copy-Item $masterSrc $masterDst
    Write-Host "✅ Backed up MASTER .env" -ForegroundColor Green
}

# Backup APPLICATION .env
$appSrc = "D:\myai\.env"
$appDst = "C:\myAI_System\backups\d-drive-configs\$timestamp-.env.application"
if (Test-Path $appSrc) {
    Copy-Item $appSrc $appDst
    Write-Host "✅ Backed up APPLICATION .env" -ForegroundColor Green
}
```

## 🎓 Best Practices

1. **Jeden MASTER .env per user**
   - Wszystkie sekrety w jednym miejscu
   - Łatwy backup i rotacja kluczy

2. **APPLICATION .env jako dokumentacja**
   - Lista wszystkich używanych zmiennych
   - Placeholdery dla sekretów

3. **AGENT .env.local tylko dla dev**
   - Szybkie testy bez zmiany głównego config
   - Usuwać po zakończeniu pracy

4. **Nigdy nie duplikować sekretów**
   - Klucz API **TYLKO** w MASTER .env
   - Inne pliki referują, nie kopiują

5. **Walidacja przy starcie agenta**
   - Sprawdzaj wymagane zmienne przed działaniem
   - Jasne error messages
