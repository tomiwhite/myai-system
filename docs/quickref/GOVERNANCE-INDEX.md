# Governance Index

Szybki punkt wejścia do dokumentów rozwojowych i produkcyjnych projektu.

## Główne dokumenty governance

- `docs/management/REPORTING-POLICY.md` - reguły klasyfikacji dokumentów i rygor raportowania po zmianach.
- `docs/management/DELIVERY-LEDGER.md` - stały rejestr zmian produkcyjnych, rozbudowy i aktualizacji projektu.

## Jak czytać dokumentację operacyjną

1. `docs/work-logs/`
- przebieg sesji i wykonane kroki techniczne

2. `docs/reports/`
- raporty wynikowe z diagnostyki, rolloutów, przeglądów ryzyk i updateów

3. `docs/management/`
- polityki governance i status delivery projektu

## Minimalny standard po zmianach

1. Nietrywialna zmiana: wpis do `docs/work-logs/`.
2. Zmiana projektowa/produkcyjna/governance: update `docs/management/DELIVERY-LEDGER.md`.
3. Wynik diagnostyczny lub rolloutowy: raport do `docs/reports/`.
